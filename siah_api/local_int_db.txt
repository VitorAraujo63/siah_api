==============================================================
  SIAH API — GUIA DE CONFIGURAÇÃO DO BANCO DE DADOS (SUPABASE)
==============================================================

Este arquivo documenta todos os locais exatos onde as credenciais
do Supabase devem ser inseridas para que a API funcione corretamente.

--------------------------------------------------------------
ARQUIVO 1: siah_api/appsettings.json
--------------------------------------------------------------

Localize a seção "Supabase" e preencha os campos:

  "Supabase": {
    "ConnectionString": "INSIRA_AQUI"
  }

Como obter a ConnectionString no Supabase:
  1. Acesse https://app.supabase.com
  2. Selecione o seu projeto
  3. Va em: Settings > Database > Connection string
  4. Escolha a opcao "URI" (modo: Transaction Pooler ou Session)
  5. Copie a string no formato:
     Host=db.XXXXXXXX.supabase.co;Database=postgres;Username=postgres;
     Password=SUA_SENHA;SSL Mode=Require;Trust Server Certificate=true

--------------------------------------------------------------
ARQUIVO 2: siah_api/appsettings.json
--------------------------------------------------------------

Localize a secao "Jwt" e preencha os campos:

  "Jwt": {
    "SecretKey": "INSIRA_AQUI"
  }

  SecretKey: Uma string longa e aleatoria (minimo 256 bits / 32 caracteres).
  Exemplo de geracao via terminal:
    openssl rand -base64 32

--------------------------------------------------------------
ARQUIVO 3: siah_api/appsettings.Development.json
--------------------------------------------------------------

Para ambiente de desenvolvimento, crie ou edite o arquivo com:

  {
    "Supabase": {
      "ConnectionString": "SUA_CONNECTION_STRING_AQUI"
    },
    "Jwt": {
      "SecretKey": "SUA_SECRET_KEY_AQUI"
    }
  }

  IMPORTANTE: Nunca commite appsettings.Development.json com credenciais reais.
  Adicione ao .gitignore se necessario.

TABELAS NECESSÁRIAS NO SUPABASE (PostgreSQL)
--------------------------------------------------------------

Execute os seguintes scripts SQL no Supabase SQL Editor:

-- Tabela principal de usuarios / pacientes
CREATE TABLE IF NOT EXISTS usuarios (
    id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    nome                  TEXT NOT NULL,
    cpf                   TEXT UNIQUE NOT NULL,
    email                 TEXT,
    telefone              TEXT,
    data_nascimento       TEXT,
    genero                TEXT,
    tipo_sanguineo        TEXT,
    hospital_vinculado    TEXT,
    rg                    TEXT,
    cartao_sus            TEXT,
    cnh                   TEXT,
    cep                   TEXT,
    rua                   TEXT,
    numero                TEXT,
    bairro                TEXT,
    cidade                TEXT,
    estado                TEXT,
    possui_plano_saude    BOOLEAN DEFAULT FALSE,
    nome_plano            TEXT,
    numero_carteirinha    TEXT,
    validade_carteirinha  TEXT,
    nome_responsavel      TEXT,
    parentesco            TEXT,
    telefone_responsavel  TEXT,
    images                TEXT[],
    embedding             FLOAT[],
    embedding_path        TEXT,
    template_biometrico   BYTEA,
    senha_hash            TEXT,
    criado_em             TIMESTAMPTZ DEFAULT NOW()
);

-- Tabela de triagens
CREATE TABLE IF NOT EXISTS triagens (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    id_usuario          UUID NOT NULL REFERENCES usuarios(id),
    queixa_principal    TEXT NOT NULL,
    peso                TEXT,
    altura              TEXT,
    temperatura         TEXT,
    pressao_arterial    TEXT,
    frequencia_cardiaca TEXT,
    criado_em           TIMESTAMPTZ DEFAULT NOW()
);

-- Tabela de especialidades medicas
CREATE TABLE IF NOT EXISTS especialidades (
    id        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    nome      TEXT NOT NULL,
    descricao TEXT
);

-- Tabela de medicos
CREATE TABLE IF NOT EXISTS medicos (
    id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    nome             TEXT NOT NULL,
    especialidade    TEXT NOT NULL,
    hospital_id      UUID,
    foto_url         TEXT,
    rating           NUMERIC(3,2) DEFAULT 0,
    disponivel_hoje  BOOLEAN DEFAULT FALSE,
    plano_ids        UUID[]
);

-- Tabela de agendamentos
CREATE TABLE IF NOT EXISTS agendamentos (
    id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    id_usuario            UUID NOT NULL REFERENCES usuarios(id),
    medico_id             UUID NOT NULL REFERENCES medicos(id),
    especialidade_id      UUID NOT NULL REFERENCES especialidades(id),
    data                  TEXT NOT NULL,
    horario               TEXT NOT NULL,
    observacoes           TEXT,
    plano_id              UUID,
    status                TEXT DEFAULT 'scheduled',
    numero_da_senha       INTEGER,
    motivo_cancelamento   TEXT,
    tipo_cancelamento     TEXT,
    reembolso_elegivel    BOOLEAN DEFAULT TRUE,
    cancelado_em          TIMESTAMPTZ,
    criado_em             TIMESTAMPTZ DEFAULT NOW()
);

-- Tabela de senhas de atendimento (fila digital)
CREATE TABLE IF NOT EXISTS senhas_atendimento (
    id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    id_paciente             UUID NOT NULL REFERENCES usuarios(id),
    totem_id                TEXT NOT NULL,
    tipo_servico            TEXT NOT NULL,
    departamento_id         UUID NOT NULL,
    numero_senha            TEXT NOT NULL,
    posicao_na_fila         INTEGER NOT NULL,
    tempo_espera_minutos    INTEGER NOT NULL,
    status                  TEXT DEFAULT 'waiting',
    emitida_em              TIMESTAMPTZ DEFAULT NOW(),
    chamada_em              TIMESTAMPTZ,
    concluida_em            TIMESTAMPTZ
);

-- Tabela de documentos clinicos (atestados, exames, receitas)
CREATE TABLE IF NOT EXISTS documentos (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    id_usuario      UUID NOT NULL REFERENCES usuarios(id),
    tipo            TEXT NOT NULL,
    titulo          TEXT NOT NULL,
    conteudo        TEXT,
    pdf_url         TEXT,
    agendamento_id  UUID REFERENCES agendamentos(id),
    ativo           BOOLEAN DEFAULT TRUE,
    criado_em       TIMESTAMPTZ DEFAULT NOW()
);

-- Tabela de tokens invalidados (logout / blacklist)
CREATE TABLE IF NOT EXISTS tokens_invalidados (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    id_usuario  UUID NOT NULL REFERENCES usuarios(id),
    token       TEXT NOT NULL,
    criado_em   TIMESTAMPTZ DEFAULT NOW()
);

-- Tabela de medicos favoritos por usuario
CREATE TABLE IF NOT EXISTS medicos_favoritos (
    id_usuario  UUID NOT NULL REFERENCES usuarios(id),
    id_medico   UUID NOT NULL REFERENCES medicos(id),
    PRIMARY KEY (id_usuario, id_medico)
);

-- Tabela de avaliacoes de medicos
CREATE TABLE IF NOT EXISTS avaliacoes_medicos (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    id_medico   UUID NOT NULL REFERENCES medicos(id),
    id_usuario  UUID NOT NULL REFERENCES usuarios(id),
    avaliador   TEXT NOT NULL,
    comentario  TEXT,
    nota        NUMERIC(2,1) NOT NULL,
    criado_em   TIMESTAMPTZ DEFAULT NOW()
);

-- Tabela de refresh tokens
CREATE TABLE IF NOT EXISTS refresh_tokens (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    id_usuario  UUID NOT NULL REFERENCES usuarios(id),
    token       TEXT NOT NULL UNIQUE,
    expira_em   TIMESTAMPTZ NOT NULL,
    criado_em   TIMESTAMPTZ DEFAULT NOW()
);

VARIÁVEL DE AMBIENTE (ALTERNATIVA AO appsettings)
--------------------------------------------------------------

Em vez de usar appsettings.json, voce pode configurar via
variaveis de ambiente no sistema operacional ou no Docker:

  Supabase__ConnectionString=Host=...
  Jwt__SecretKey=...

--------------------------------------------------------------
INTEGRAÇÃO BIOMETRIA (DPFP / DigitalPersona)
--------------------------------------------------------------

O endpoint POST /api/biometria/identificar usa a biblioteca
DigitalPersona Fingerprint SDK (DPFP). Para habilitar:

  1. Obtenha o SDK da DigitalPersona:
     https://www.crossmatch.com/software/
  2. Adicione a DLL ao projeto
  3. Substitua o stub em:
     Infrastructure/Services/FaceEmbeddingServiceStub.cs
  4. Implemente a interface:
     Domain/Ports/Input/IBiometriaMatchService.cs

--------------------------------------------------------------
INTEGRAÇÃO RECONHECIMENTO FACIAL (VGG-Face / DeepFace)
--------------------------------------------------------------

O endpoint POST /api/pacientes/reconhecer compara embeddings
armazenados no banco. A geracao de novos embeddings a partir
das imagens Base64 requer um servico ML externo (Python/DeepFace).

  Arquivo de stub:
    Infrastructure/Services/FaceEmbeddingServiceStub.cs

  Para integrar o servico Python DeepFace:
    Configure a URL em appsettings.json:
      "FaceEmbedding": {
        "ServiceUrl": "http://localhost:5001/generate-embedding"
      }
