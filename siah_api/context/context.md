## "SIAH_Especificacao_API_v1"
### 1. Visão GeralO SIAH é um sistema hospitalar integrado que conecta pacientes, médicos e unidades de saúde por meio de um aplicativo mobile. Esta API RESTful é o backbone de todas as operações do app, sendo responsável por autenticação, gestão de perfil, agendamento de consultas, histórico médico, documentos clínicos e o diferencial principal do sistema: a entrega de senhas de atendimento diretamente no celular do paciente validado no totem.1.1 Informações GeraisBase URL (Produção): [https://api.siah.com.br/v1](https://api.siah.com.br/v1)Base URL (Staging): [https://staging.api.siah.com.br/v1](https://staging.api.siah.com.br/v1)Protocolo: HTTPS obrigatório em todos os ambientesFormato de Dados: application/json (UTF-8)Autenticação: JWT Bearer Token (Authorization: Bearer {token})Versioning: Via prefixo de rota: /v1/, /v2/...Timezone: America/Sao_Paulo (UTC-3)1.2 Autenticação e SegurançaTodas as rotas, exceto login e cadastro, exigem autenticação via JWT. O token deve ser enviado no header de cada requisição:HEADER: Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...Tempo de expiração do Access Token: 60 minutosTempo de expiração do Refresh Token: 30 diasRefresh automático via endpoint dedicado, sem necessidade de novo loginTokens inválidos ou expirados retornam HTTP 401Após 5 tentativas de login falhas, conta bloqueada por 15 minutos1.3 Padrão de RespostasTodas as respostas seguem um envelope padronizado:HTTP StatusSignificadoUso Típico200 OKSucessoGET, PUT, PATCH bem-sucedidos201 CreatedRecurso criadoPOST bem-sucedido (cadastro, agendamento)400 Bad RequestDados inválidosCampos faltando ou formato errado401 UnauthorizedToken inválidoJWT ausente, expirado ou revogado403 ForbiddenSem permissãoAcesso a recurso de outro usuário404 Not FoundNão encontradoRecurso não existe409 ConflictConflitoCPF já cadastrado, horário ocupado422 UnprocessableEntidade inválidaSenhas não conferem, CPF inválido429 Too Many Req.Rate limitMuitas requisições em pouco tempo500 Server ErrorErro internoFalha inesperada no servidor2. Senha de Atendimento — Totem Digital🏆 FUNCIONALIDADE PRINCIPALEste é o diferencial central do SIAH. Ao invés de imprimir uma senha em papel no totem, o sistema envia um popup interativo diretamente ao celular do paciente autenticado, com posição na fila, estimativa de espera e alertas em tempo real.5.1 Fluxo da Senha DigitalPaciente chega ao hospital e se aproxima do totemRealiza validação no totem (CPF + reconhecimento facial ou QR Code do app)Totem comunica com a API via endpoint seguro, informando paciente e tipo de atendimentoAPI gera a senha, associa ao paciente e dispara push notification + WebSocket ao appPaciente recebe popup no celular com número, fila e previsão de esperaConforme a fila avança, o app atualiza em tempo real via WebSocketQuando chamado, novo push notification é enviado com alerta sonoro5.2 Endpoints de SenhaMétodoEndpointDescriçãoPOST/queue/validate-totemTotem valida paciente e emite senha (uso interno/totem)GET/queue/my-ticketPaciente consulta sua senha activa atualGET/queue/status/:ticket_idStatus em tempo real de uma senha específicaPOST/queue/confirm-arrivalPaciente confirma presença ao ser chamadoWS/ws/queue/:ticket_idWebSocket para atualizações em tempo real da filaPOST /queue/validate-totem — Emissão de Senha (Totem → API)Este endpoint é chamado pelo sistema do totem, não pelo app do paciente. Requer autenticação via API Key exclusiva do totem.REQUEST BODY: patient_cpf | patient_id (UUID) | totem_id | service_type: 'consultation' | 'exam' | 'emergency' | 'pharmacy' | department_idRESPONSE 201: { success: true, data: { ticket_id, ticket_number: 'A-042', queue_position: 7, estimated_wait_minutes: 28, department: 'Pediatria', issued_at } }⚡ AÇÃO AUTOMÁTICA: Ao emitir a senha, a API dispara automaticamente: (1) Push Notification para o app do paciente, (2) Evento WebSocket no canal do paciente, (3) Registro no histórico de atendimentos.GET /queue/my-ticket — Minha Senha AtivaRESPONSE 200: { success: true, data: { ticket_id, ticket_number, status: 'waiting' | 'called' | 'in_service' | 'completed' | 'missed', queue_position, estimated_wait_minutes, called_at: null, department } }WS /ws/queue/:ticket_id — Tempo RealConexão WebSocket mantida pelo app durante o período de espera.
